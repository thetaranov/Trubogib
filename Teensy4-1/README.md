# Teensy 4.1 — Контроллер движения трубогиба

## Статус: ✅ Макетная плата собрана | Прошивка v2.0

Макетная плата из 4× MAX490 + 1× MAX485 собрана, готова к подключению в шкаф.

## Назначение

Teensy 4.1 — контроллер движения, заменяющий ESP8266.
- Принимает данные от ПЛК Owen PLC110-30 по **Modbus RTU** (RS-485, Slave ID=1)
- Генерирует **Step/Dir** импульсы с трапецеидальным профилем для двух серводрайверов MR-JE-200A
- Управляет процедурой **Home** для осей Z и C (с антидребезгом 20 мс)
- Принимает команды от **EXE TubeBender** через **USB-Serial**
- **Watchdog:** потеря Modbus-связи >2 сек → аварийная остановка + снятие SON

## Оборудование

| Компонент | Модель |
|-----------|--------|
| Контроллер | Teensy 4.1 (ARM Cortex-M7, 600 MHz) |

## Аварийные сигналы серво (ALM) — D17/D18

Серводрайверы MR-JE-200A выдают сигнал ALM через CN1 pin 37 (бело-голубой провод, b-contact NC):

- **Норма:** транзистор ON → DI ПЛК = TRUE → almZ/almC = 0
- **Авария (AL.xx):** транзистор OFF → DI ПЛК = FALSE → almZ/almC = 1

Входы ПЛК:
- **D17** (`gDI_17 AT %IX1.1.6`) — Servo Z ALM
- **D18** (`gDI_18 AT %IX1.1.7`) — Servo C ALM

ПЛК инвертирует сигнал (`almZ := NOT gDI_17`) и передаёт флаги через **Modbus регистр 7** (bit0=almZ, bit1=almC).

Teensy читает регистр 7 и:
- Добавляет `"alm":{"z":0,"c":0}` в JSON-статус
- При обнаружении аларма: останавливает оси, снимает SON

## USB-Serial протокол (для EXE TubeBender)

115200 бод, команды завершаются `\n`:

| Команда | Описание |
|---------|----------|
| `?` | JSON-статус (позиции, Modbus, ПЛК) |
| `REGS` | Дамп всех регистров |
| `MZ pos,spd` | Движение Z на pos мм со скоростью spd мм/с |
| `MC pos,spd` | Движение C на pos ° со скоростью spd °/с |
| `HZ` / `HC` | Home ось Z / C |
| `STOP` | Остановить обе оси |
| `ESTOP` | Аварийная остановка + SON off |
| `SON 1` / `SON 0` | Servo ON / OFF |
| `CMD n` | Записать gExtCommand (рег.9) |
| `TGT n` | Записать gExtTarget (рег.10) |
| `WR reg val` | Записать в регистр |
