## 1. Owen PLC110-30 — программа CoDeSys

### 1.1. Общие сведения

| Параметр | Значение |
|----------|----------|
| Контроллер | Owen PLC110-30 [M02] |
| Версия | **PLC110-24.30.Р-М** (релейные выходы) |
| Среда | CoDeSys v2.3.9 |
| Язык | Structured Text (ST) |
| Файл проекта | `ПЛК/TRUBOGIB.pro` |
| Экспорт программы | `ПЛК/TRUBOGIB_CLEAN.exp` |
| Target-файлы | `ПЛК/Таргет_Файлы_PLC110.30_m_v2/` |
| Связь (программирование) | USB или Ethernet |
| Связь (рабочая) | RS-485 Modbus RTU Master |
| Цикл ПЛК | T ≈ 10 мс |

> **Выходы:** DO7, DO8, DO10, DO11 — **релейные**. DO7 = SON Z, DO8 = SON C (синие провода CN1), DO10/DO11 = мотор гибки.

### 1.2. Привязка I/O

**Быстрые входы (HSC) — энкодер оси гибки:**

| Адрес | Клемма | Сигнал |
|-------|--------|--------|
| `%IX0.0` | — | Энкодер A (HSC канал A) |
| `%IX0.1` | — | Энкодер B (HSC канал B) |

Настройка HSC:
```
Fast Inputs → Channel 0:
  Mode:    Quadrature A/B (4x)
  Preset:  0
  Reset:   программный
```

**Дискретные входы:**

| Адрес | Клемма | Сигнал |
|-------|--------|--------|
| `%IX1.4` | D5 | Переключатель режима (ручной/авто) |
| `%IX1.5` | D7 | Педаль вперёд (гибка +) |
| `%IX1.6` | D8 | Педаль назад (гибка −) |

**Дискретные выходы (релейные):**

| Адрес | Клемма | Сигнал |
|-------|--------|--------|
| `%QX3.0.2` | DO7 | SON пара 1 — Servo ON ось Z (синий провод CN1) |
| `%QX3.0.5` | DO10 | Мотор вперёд (гибка +) |
| `%QX3.0.6` | DO11 | Мотор назад (гибка −) |
| `%QX3.0.3` | DO8 | SON пара 2 — Servo ON ось C (синий провод CN1) |

### 1.3. Глобальные переменные (GVL)

| Переменная | Тип | RETAIN | Описание |
|------------|-----|--------|----------|
| `gCurrentAngle` | INT | ✅ | Текущий угол гибки (°), 1 импульс = 1° |
| `gTargetAngle` | INT | ✅ | Целевой угол для автоматического режима |
| `gDefaultTarget` | INT | ✅ | Угол по умолчанию при старте цикла 1 |
| `gCycleNum` | INT | ✅ | Номер цикла: 1 = гибка вперёд, 2 = возврат |
| `gMode` | BOOL | ❌ | Режим: FALSE = ручной, TRUE = авто |
| `gDone` | BOOL | ❌ | Признак достижения цели |
| `gBeeper` | BOOL | ❌ | Управление бипером (отдельный выход %QX5.0) |
| `sonZ` | BOOL | ❌ | SON пара 1 (ось Z) → DO7 |
| `sonC` | BOOL | ❌ | SON пара 2 (ось C) → DO8 |

### 1.4. Алгоритм основного цикла PLC_PRG

```
КАЖДЫЙ ЦИКЛ ПЛК (T ≈ 10 мс):

1. Чтение входов:
   gMode         ← %IX1.4
   pedalFwd      ← %IX1.5
   pedalRev      ← %IX1.6
   gCurrentAngle ← HSC_value

2. Обработка Modbus-команд:
   IF gExtCommand = 1 THEN
     HSC_value := 0
     gCurrentAngle := 0
     gExtCommand := 0
   END_IF
   IF gExtTarget > 0 THEN
     gTargetAngle := gExtTarget
     gExtTarget := 0
   END_IF

3. IF gMode = FALSE (ручной):
   pedalFwd → %QX3.0.0 (мотор+)
   pedalRev → %QX3.0.1 (мотор−)

4. IF gMode = TRUE (авто):
   IF gCycleNum = 1:
     Разрешена только педаль D7 (вперёд)
     IF gCurrentAngle >= gTargetAngle THEN
       gDone := TRUE
       gBeeper := TRUE
       → таймер 500 мс
       → gCycleNum := 2
       → gTargetAngle := 0
     END_IF

   IF gCycleNum = 2:
     Разрешена только педаль D8 (назад)
     IF gCurrentAngle <= gTargetAngle THEN
       gDone := TRUE
       gBeeper := TRUE
       → таймер 500 мс
       → gCycleNum := 1
       → gTargetAngle := gDefaultTarget
     END_IF

5. Servo ON/OFF (SON):
  CMD 6 → sonZ:=TRUE, sonC:=TRUE → DO7:=TRUE, DO8:=TRUE
  CMD 7 → sonZ:=FALSE, sonC:=FALSE → DO7:=FALSE, DO8:=FALSE
   CMD 10 (E-Stop) → SON OFF + остановка мотора

6. Modbus: запись регистров 0–6 каждые 100 мс
            чтение регистров 9–10 каждые 100 мс
```

> **Проверка цели:** направленная (`>=` / `<=`), не `ABS`
> **Блокировка:** противоположная педаль блокируется по номеру цикла
> **Расчёт угла:** `gCurrentAngle = HSC_value` (прямое присвоение)
> **HSC формула:** 360 P/R × 4x квадратура = 1440 отсчётов/оборот (в прошивке упрощено: 1 импульс ≈ 1°)

### 1.5. Калибровка (обнуление)

**Метод 1 — через EXE v2.1:**
1. Нажать кнопку «ОБНУЛИТЬ» в интерфейсе EXE
2. EXE отправляет команду через шлюз, который записывает `gHostCommand = 1` в регистр 9
3. ПЛК: `HSC_value := 0`, `gCurrentAngle := 0`

**Метод 2 — аппаратный Z-mark:**
- Подключить Z-выход энкодера ко входу ПЛК → автосброс HSC при проходе метки

### 1.6. Типичные ошибки CoDeSys при работе с ПЛК

| Ошибка | Причина | Решение |
|--------|---------|---------|
| `C0001: Identifier not declared` | Переменная не объявлена | Добавить в VAR...END_VAR (верхняя панель) |
| `C0032: Type mismatch` | INT vs BOOL | `INT_TO_BOOL()` / `BOOL_TO_INT()` |
| HSC не считает | Не привязаны быстрые входы | PLC Configuration → Fast Inputs |
| Modbus нет связи | Неверный COM/скорость | 115200 8N1, проверить A/B RS-485 |
| RETAIN не работает | Переменная без RETAIN | Добавить `VAR RETAIN` |

## 2. Протокол Modbus RTU

### 2.1. Физический уровень

| Параметр | Значение |
|----------|----------|
| Интерфейс | RS-485 (полудуплекс, витая пара) |

| Формат | **8N1** (8 бит данных, без чётности, 1 стоп-бит) |

| Устройство | Роль | Адрес |

### 2.2. Актуальная карта регистров (Holding Registers)

> **ВАЖНО:** Документ `04_MODBUS_ПРОТОКОЛ.md` содержал ошибку в порядке регистров.
> Правильная карта верифицирована по **коду ПЛК (TRUBOGIB_CLEAN.exp)** и **коду ESP8266**.

**Регистры записи ПЛК → Slave (FC 0x10), 7 регистров каждые 100 мс:**

| Регистр | Переменная ПЛК | Тип | Описание |
|---------|----------------|-----|----------|
| **0** | `gCurrentAngle` | INT16 | **Текущий угол гибки (°)** |
| **1** | `gTargetAngle` | INT16 | **Целевой угол (°)** |
| **2** | `gCycleNum` | INT16 | Номер цикла (1 или 2) |
| **3** | `gMode` | INT16 | Режим: 0 = ручной, 1 = авто |
| **4** | `gDone` | INT16 | Достижение цели: 0/1 |
| **5** | `gPedalFwd` | INT16 | Педаль вперёд: 0/1 |
| **6** | `gPedalRev` | INT16 | Педаль назад: 0/1 |

**Регистры чтения Slave → ПЛК (FC 0x03), 2 регистра каждые 100 мс:**

| Регистр | Переменная | Тип | Описание |
|---------|------------|-----|----------|
| **9** | `gExtCommand` | INT16 | Команда: 0 = нет, 1 = обнулить угол |
| **10** | `gExtTarget` | INT16 | Целевой угол от внешнего управления |

> Совместимость: прошивка `Teensy4_Trubogib.ino` дублирует `gExtCommand` также в регистр **8** (legacy-карта `0..9`), чтобы исключить ошибки смещения адресов в старых конфигурациях ПЛК.
>
> Для надёжного срабатывания edge-логики ПЛК команда в `gExtCommand` передаётся стадийно: `0` (≈250 мс) → `CMD` (≈1500 мс) → `0` (≈500 мс).

**Расширенная карта (для интеграционного этапа):**

| Регистр | Имя | Направление | Описание |
|---------|-----|-------------|----------|
| 0 | DefaultTarget | ПЛК → ESP | Угол по умолчанию |
| 1 | CurrentAngle | ПЛК → ESP | Текущий угол (энкодер) |
| 2 | TargetAngle | ПЛК → ESP | Целевой угол |
| 3 | PlcState | ПЛК → ESP | 0=Idle, 1=Bending, 2=Returning, 3=Done, 4=Error |
| 4 | PlcAlarms | ПЛК → ESP | Битовая маска аварий |
| 5 | StepIndex | ПЛК → ESP | Номер текущего шага программы |
| 6 | CycleCount | ПЛК → ESP | Счётчик выполненных циклов |
| 7 | Reserved | — | — |
| 8 | CmdRegister | ESP → ПЛК | 0=None, 1=Start, 2=Stop, 3=Home, 4=E-Stop |
| 9 | CmdTargetAngle | ESP → ПЛК | Целевой угол от EXE |
| 10 | CmdStepIndex | ESP → ПЛК | Номер шага для выполнения |
| 11–15 | Reserved | — | — |

### 2.3. Форматы кадров Modbus RTU

**Запись (FC 0x10 — Write Multiple Registers):**
```
[Addr] [FC] [StartH] [StartL] [CountH] [CountL] [ByteCount] [Data...] [CRCH] [CRCL]
  01     10     00       00       00       07        0E       [14 bytes]  XX    XX

Addr:       0x01 (slave address)
FC:         0x10 (Write Multiple Holding Registers)
Start:      0x0000 (регистр 0)
Count:      0x0007 (7 регистров)
ByteCount:  0x0E (14 байт = 7 × 2)
Data:       7 × INT16 (big-endian)
CRC:        CRC16 Modbus (little-endian)
```

**Чтение (FC 0x03 — Read Holding Registers):**
```
[Addr] [FC] [StartH] [StartL] [CountH] [CountL] [CRCH] [CRCL]
  01     03     00       09       00       02       XX    XX

Start:  0x0009 (регистр 9)
Count:  0x0002 (2 регистра)
```

**Ответ на чтение:**
```
[Addr] [FC] [ByteCount] [Data...] [CRCH] [CRCL]
  01     03      04      [4 bytes]   XX    XX

Data[0-1]: gExtCommand (big-endian)
Data[2-3]: gExtTarget  (big-endian)
```

### 2.4. CRC16 Modbus

```
Полином: 0xA001 (отражённый 0x8005)
Начальное значение: 0xFFFF
Порядок байт в кадре: младший байт первым (little-endian)
```

**Алгоритм (псевдокод):**
```
crc = 0xFFFF
for each byte in frame (excluding CRC bytes):
    crc ^= byte
    for i = 0 to 7:
        if (crc & 1):
            crc = (crc >> 1) ^ 0xA001
        else:
            crc = crc >> 1
append crc_low, crc_high to frame
```

**Реализация на C++ (Teensy):**
```cpp
uint16_t crc16(uint8_t* buf, int len) {
    uint16_t crc = 0xFFFF;
    for (int i = 0; i < len; i++) {
        crc ^= buf[i];
        for (int j = 0; j < 8; j++) {
            if (crc & 1) crc = (crc >> 1) ^ 0xA001;
            else crc >>= 1;
        }
    }
    return crc;
}
```

### 2.7. Типичные проблемы Modbus

| Симптом | Причина | Решение |
|---------|---------|---------|

## 3. Teensy — legacy-контроллер (устарело)

### 3.1. Общие сведения

| Параметр | Значение |
|----------|----------|
| Плата | NodeMCU Teensy (ESP-12E) |
| MCU | Tensilica Xtensa L106 @ 80 МГц |
| RS-485 | SoftwareSerial → MAX485 |
| Статус | ⛔ **Legacy — не используется**, заменён на Teensy 4.1 |

### 3.2. Назначение (Историческое)

1. **Modbus RTU Slave** (addr=1) — получал данные от ПЛК, отправлял команды.
2. Использовался как отладочный шлюз с WiFi/HTTP интерфейсом. **WiFi более не используется** — это было нужно только для отладки. Всё управление станком теперь происходит или с физических кнопок в ручном режиме, или через EXE-программу + Teensy 4.1 в автоматическом режиме по Modbus RTU.

> **Ограничение:** Teensy **не способен** генерировать Step/Dir для сервоприводов (нет нужных аппаратных таймеров). Оси Z и C управлялись только виртуально (Dry Run).

### 3.3. Схема подключения Teensy → MAX485 → ПЛК

```
Teensy (NodeMCU)            MAX485              Owen PLC110-30
─────────────────            ──────              ──────────────
D6 (GPIO12) ──► SoftSerial TX ──► DI
D5 (GPIO14) ◄── SoftSerial RX ◄── RO
D7 (GPIO13) ──► DE + RE (direction)
                               A ◄──────────────► A (RS-485)
                               B ◄──────────────── B (RS-485)
3V3 ──────────────────────────► VCC MAX485
GND ──────────────────────────► GND
```

### 3.4. Особенности реализации

- **SoftwareSerial** для RS-485 (аппаратный UART занят USB-отладкой)
- **Разделение склеенных кадров** — если ПЛК отправляет 2 кадра без паузы, прошивка разделяет по длине
- **CRC16** — полная проверка входящих и генерация для исходящих
- **Non-blocking** — цикл работы не блокирует основную программу

### 4.3. Требования к логике (Алгоритм прошивки)

1. **Библиотеки:** Использовать `TeensyStep` или `AccelStepper`.
2. **Частота импульсов:** До 150 кГц (максимум) для двух осей, 10 000 импульсов/оборот.
3. **Профиль движения:** Трапецеидальный (плавный разгон и торможение).
4. **Modbus RTU:** Slave ID = 1, Baud Rate **115200**, формат **8N1**.
5. **Безопасность:**
   - Безопасная аппаратная/программная отработка E-Stop (остановка импульсов и снятие SON).
   - Поддержка "Dry Run" (виртуальный прогон без фактического вращения моторов).
6. **Датчики Home:** Программный антидребезг (Debounce) для NPN датчиков Home_Z и Home_C.
7. **Управление:** Работа без встроенного HTTP-окружения. Прием команд и выдача статусов строго через Modbus от управляющей EXE (либо физические кнопки ПЛК в ручном режиме).

### 4.4. Генерация Step/Dir (Motion Control)

```cpp
void stepMotor(int stepPin, int dirPin, long steps, int delayUs) {
    digitalWrite(dirPin, steps > 0 ? HIGH : LOW);
    steps = abs(steps);
    for (long i = 0; i < steps; i++) {
        digitalWrite(stepPin, HIGH);
        delayMicroseconds(delayUs / 2);
        digitalWrite(stepPin, LOW);
        delayMicroseconds(delayUs / 2);
    }
}
```

**Трапецеидальный профиль скорости:**
```
Параметры (в прошивке Teensy):
  fStart = 1000 Гц      (стартовая частота)
  fMax   = 100000 Гц    (максимальная частота, 100 кГц)
  accel  = 50000 Гц/с   (время разгона ~2 сек)

Фазы:
  1. Разгон:   fStart → fMax
  2. Постоянная скорость: fMax
  3. Торможение: fMax → fStart
```

**Home-процедура оси Z:**
```
1. Включить SON (серво ON)
2. Двигаться медленно в −Z (к Home-датчику)
3. При срабатывании Home → остановить
4. Отъехать на 5 мм в +Z
5. Снова медленно в −Z
6. При срабатывании → позиция = 0
7. Отъехать в рабочую зону
```

## 5. Шпаргалка CoDeSys v2.3.9

### 5.1. Установка и создание проекта

1. Скачать CoDeSys v2.3.9 с сайта Owen (owencloud.ru)
2. Скопировать `PLC110.30_m_v2/` в `C:\Program Files\...\CoDeSys\Targets\`
3. Запустить `InstallTarget.bat`
4. Файл → Новый → Target: **PLC110-30 [M02]** → Язык: **Structured Text (ST)**

### 5.2. Двухпанельный редактор (⚠️ КРИТИЧНО!)

```
┌──────────────────────────┐
│  Верхняя панель:          │  ← VAR...END_VAR (объявления)
│  VAR                     │
│    x : INT;              │
│  END_VAR                 │
├──────────────────────────┤
│  Нижняя панель:           │  ← тело программы (ST-код)
│  x := x + 1;            │
│  IF x > 10 THEN ...     │
└──────────────────────────┘
```

> При вставке импортированного кода: `VAR...END_VAR` → в верхнюю панель, остальное → в нижнюю!

### 5.4. Типы данных

| Тип | Размер | Диапазон |
|-----|--------|----------|
| BOOL | 1 бит | TRUE/FALSE |
| INT | 16 бит | −32768…32767 |
| DINT | 32 бит | −2³¹…2³¹−1 |
| UINT | 16 бит | 0…65535 |
| REAL | 32 бит | IEEE 754 |
| TIME | 32 бит | T#0ms…T#49d |

**Преобразование типов:**
```pascal
INT_TO_REAL(x)    (* INT → REAL *)
REAL_TO_INT(x)    (* REAL → INT *)
INT_TO_BOOL(x)    (* INT → BOOL, 0=FALSE *)
BOOL_TO_INT(x)    (* BOOL → INT, TRUE=1 *)
```

### 5.5. Таймеры и триггеры

**TON (задержка включения):**
```pascal
VAR  tonDelay : TON;  END_VAR
tonDelay(IN := condition, PT := T#500ms);
IF tonDelay.Q THEN  (* прошло 500 мс *)  END_IF
```

**R_TRIG (нарастающий фронт):**
```pascal
VAR  rtPedal : R_TRIG;  END_VAR
rtPedal(CLK := pedalInput);
IF rtPedal.Q THEN  (* одиночный импульс *)  END_IF
```

**RETAIN-переменные:**
```pascal
VAR RETAIN
    gCurrentAngle : INT := 0;
    gCycleNum     : INT := 1;
    gDefaultTarget: INT := 90;
END_VAR
```

### 5.6. Импорт программы из .exp файла

1. Файл → Новый проект → Target: PLC110-30 [M02]
2. Проект → Импорт → выбрать `TRUBOGIB_CLEAN.exp`
3. **Важно:** CoDeSys v2.3 — двухпанельный редактор, VAR...END_VAR строго в верхнюю панель
4. Проект → Перестроить всё (Ctrl+F11)
5. PLC Configuration → Fast Inputs → привязать `%IX0.0`, `%IX0.1`
6. Discrete Inputs → привязать `%IX1.4`, `%IX1.5`, `%IX1.6`
7. Discrete Outputs → привязать `%QX3.0.2` (DO7 SON Z), `%QX3.0.3` (DO8 SON C), `%QX3.0.5` (DO10), `%QX3.0.6` (DO11), `%QX5.0` (Beeper)
8. Онлайн → Параметры связи → USB (115200) или Ethernet
9. Онлайн → Загрузить → Старт (F5)
