## 3. Архитектура UI: FBO + ImGui Docking

### 3.1 Принцип

```
1. 3D-сцена рендерится в FBO (Framebuffer Object) — off-screen текстура
2. Текстура FBO отображается внутри ImGui-окна через ImGui::Image()
3. 3D-контент строго ограничен рамками своего окна
4. Все окна управляются через ImGui Docking — единая система привязки
```

Этот же подход используют: FL Studio, Blender, Unity Editor, Unreal Editor.

**Почему НЕ фоновый рендер:** 3D вылезает за границы, нет multi-viewport, проблемы со scissor.

### 3.2 Pipeline рендеринга (порядок вызовов)

```
1. app.update()          — логика, анимация
2. app.renderToFBO()     — 3D → FBO (glBindFramebuffer → render → glBindFramebuffer(0))
3. glClear(screen)       — очистка экрана
4. app.drawUI()          — ImGui (включая Image() с FBO текстурой)
5. ImGui::Render()       — финализация
6. glfwSwapBuffers()
```

### 3.3 FBO-методы Renderer

```cpp
void ensureFBO(int w, int h);         // Создание/пересоздание FBO нужного размера
void renderToFBO();                    // Рендер сцены в FBO
unsigned int fboTextureId() const;     // Texture ID для ImGui::Image
int fboWidth() const;
int fboHeight() const;
```

Каждый кадр: если размер content-области окна «Сцена» изменился → пересоздать FBO (RGBA8 + DEPTH24_STENCIL8).

## 4. Система координат

| Ось | Направление |
|-----|-------------|
| +Y | От камеры к станку |
| −Y | От станка к камере (направление подачи трубы) |
| +X | Вправо на экране (направление гиба) |
| Z | Вертикальная ось сцены |

- **Point0** = (−76, 0, 347) мм — точка начала пластической деформации (из STP-файла)
- **AxisZ_bender** = (0, 0, 1) — ось вращения гибочного узла (проходит через начало координат)
- Начальная камера: yaw = −90°, pitch = 25°, dist = 800, center ≈ (−76, −100, 347)
- При подаче труба **выезжает из станка в сторону камеры** (в −Y)

## 5. Геометрия и кинематика гибки

### 5.1 Термины и обозначения

| Термин | Описание |
|--------|----------|
| **D** | Наружный диаметр трубы (мм), глобальный, единый для всей программы |
| **R (CLR)** | Center Line Radius — радиус гиба по средней линии (мм), глобальный |
| **ClampLength** | Длина оснастки прижима (мм), по умолчанию 60 мм |
| **L_feed** | Подача — линейное перемещение заготовки (мм) |
| **Rotation** | Поворот трубы вокруг продольной оси (°), знак по правилу правой руки |
| **Angle** | Угол гиба (°) |
| **L_bend** | Длина дуги: `L_bend = π × R × Angle / 180` |
| **L_total** | Полная развёртка: `L_total = Σ(L_feed_i) + Σ(L_bend_i)` |
| **Point0** | Фиксированная точка начала деформации на станке |

### 5.2 Ограничения входных параметров

- `D > 0`, `R > 0`, `ClampLength >= 0`
- `L_feed_i >= 0`; если два соседних шага имеют `Angle > 0`, то `L_feed_i >= ClampLength`
- `0 <= Angle_i <= 180`
- `Rotation_i ∈ [−360; +360]`
- `D` и `R` едины для всех шагов; смешивание внутри программы запрещено

### 5.3 Цветовое кодирование зон трубы

| Цвет | Зона |
|------|------|
| **Синий** | Материал до Point0 (ещё не прошёл зону гиба) |
| **Красный** | Прямой участок после Point0 |
| **Зелёный** | Дуга (гиб) после Point0 |

**Инвариант:** `L_синий + L_красный + L_зелёный = L_total` — выполняется на каждом кадре.

### 5.4 Кинематика — правила движения

**Подача (Feed):**
- Новый красный сегмент вырастает из Point0 в −Y длиной `L_feed`
- Вся ранее сформированная цепь (предыдущие красные + зелёные) сдвигается в −Y на `L_feed`
- Синяя часть уменьшается: `L_blue -= L_feed`
- Прижим открыт во время подачи

**Ротация:**
- Поворот фрейма вокруг текущего `fwd` на `Rotation` — меняет плоскость следующего гиба
- Геометрия и длины не изменяются; прижим открыт

**Гибка:**
- Ролик физически фиксирован; все гибы строго в плоскости XY
- Зелёная дуга вырастает из конца нового красного и толкает его по траектории дуги R
- `L_blue -= L_bend`; прижим зажат во время гибки

**Ключевое правило:** ролик фиксирован — ротация меняет ориентацию последующих сегментов, но не плоскость самого гиба.

## 6. Алгоритм обратного обхода (buildTubeGeometry)

Геометрия строится обходом шагов в **обратном порядке** (от последнего к первому):

```text
pos   = Point0
fwd   = (0, −1, 0)   // −Y: машинное направление подачи
up    = (0,  0, 1)   // Z вверх
right = cross(up, fwd) = (+1, 0, 0)  // +X: направление гиба

for i from N−1 downto 0:
    // 1. Гиб (ВСЕГДА в XY — ролик фиксирован, гиб ДО ротации)
    if step[i].Angle > 0:
        Draw Green arc from pos, radius R, in plane (fwd, right), angle = Angle_i
        update pos/fwd/up/right to arc end

    // 2. Ротация (влияет на ориентацию последующих сегментов)
    if step[i].Rotation != 0:
        rotate (up, right) around fwd by Rotation_i

    // 3. Подача
    if step[i].LFeed > 0:
        Draw Red segment from pos in fwd direction, length = LFeed_i
        pos += fwd * LFeed_i
```

**Почему обратный порядок:** каждая новая подача толкает сформированную часть в −Y. Обратный обход автоматически даёт правильное смещение — последний шаг ближе к Point0, первый — дальше от станка.

**Почему гиб ДО ротации:** ролик фиксирован в XY; ротация в обратном обходе означает «твист для предшествующих сегментов».

### Псевдокод верхнего уровня (рендер)

```text
Precompute: L_total = Σ(LFeed_i) + Σ(LBend_i)

// Синий хвост (необработанный материал)
Draw Blue cylinder from Point0 toward +Y, length = L_total − consumed

// Обратный обход
pos = Point0; fwd = (0,−1,0); up = (0,0,1); right = (+1,0,0)
for i from N−1 downto 0:
    if Angle_i > 0:
        Draw Green arc from pos, R, plane(fwd,right), Angle_i
        update pos/fwd/up/right
    if Rotation_i != 0:
        rotate (up, right) around fwd by Rotation_i
    if LFeed_i > 0:
        Draw Red segment from pos in fwd, LFeed_i
        pos += fwd * LFeed_i
```

### Контрольный пример (D=38, R=76)

Программа: Шаг 1 — L=100, A=90°; Шаг 2 — L=200, A=0°

`L_total = 100 + π×76×90/180 + 200 ≈ 419.38 мм`

| Элемент | Координаты |
|---------|-----------|
| Red2 (шаг 2) | от (−76, 0, 347) до (−76, −200, 347), длина 200мм, в −Y |
| Green1 (шаг 1) | дуга от (−76, −200, 347) до (0, −276, 347), центр (0, −200, 347), R=76, 90° |
| Red1 (шаг 1) | от (0, −276, 347) до (100, −276, 347), длина 100мм, в +X |

### Требования к гладкости дуги

- Угловой шаг аппроксимации: `Δφ ≤ 6.67°` или длина хорды по средней линии `≤ 2 мм`
- G1-непрерывность (касательная в стыке прямой/дуга)
- Разрешение меша: 12 сегментов поперечного сечения (вместо 16 — на 25% легче)

## 7. Анимация гибки (7 фаз на шаг)

Порядок фаз: **A → B → B2 → C → D → E → F**. Каждая фаза завершается полностью перед началом следующей. Если `Angle = 0`, фазы B2–F пропускаются.

| Фаза | Название | Скорость | Прижим | STL ролика |
|------|----------|----------|--------|------------|
| A | **Feed (подача)** | 300 мм/с | Открыт | Неподвижен (0°) |
| B | **Rotation (ротация)** | 120°/с | Открыт | Неподвижен (0°) |
| B2 | **Зажим прижима** | 600 мм/с, ход 150 мм | Закрывается | Неподвижен (0°) |
| C | **Bend (гибка)** | 60°/с | Зажат | Вращается 0° → Angle |
| D | **Отвод прижима** | 600 мм/с | Открывается | Неподвижен на Angle |
| E | **Отвод трубы (clearance)** | 300 мм/с, ClampLength мм | Открыт | Неподвижен на Angle |
| F | **Возврат ролика** | 120°/с | Открыт | Вращается Angle → 0° |

**Важно:** Порядок D→E строго обязателен. Прижим ДОЛЖЕН отъехать до подачи трубы вперёд, иначе трубу закусит.

**Соответствие benderAngle:**
```
Feed:         benderAngle = 0
Rotation:     benderAngle = 0
Clamp:        benderAngle = 0
Bend:         benderAngle = step.Angle * progress     (0→1)
RelClamp:     benderAngle = step.Angle
TubePush(E):  benderAngle = step.Angle
Return(F):    benderAngle = step.Angle * (1−progress) (1→0)
```

**Подсветка в таблице:** текущая строка шага выделена; ячейка текущей фазы подсвечена светло-синим. Прогресс-бар с названием фазы виден всегда.

## 8. Лазерная резка — 10-фазная анимация

### 8.1 Позиционирование лазерной головки

Лазерная головка закреплена стационарно:
```
LaserPoint.X = Point0.X = −76
LaserPoint.Y = Point0.Y − LaserOffsetY = 0 − 60 = −60 мм
LaserPoint.Z = Point0.Z + D/2 + LaserGapZ = 347 + 19 + 5 = 371 мм (для D=38)
```
Луч направлен вертикально вниз (0, 0, −1). Точка контакта с трубой — верхняя образующая (θ=0°).

Параметры головки:

| Параметр | По умолч. | Описание |
|----------|-----------|----------|
| `LaserOffsetY` | 60 мм | Смещение головки по −Y от Point0 |
| `LaserGapZ` | 5 мм | Зазор от верхней образующей трубы до сопла |
| `WhipLength` | 6000 мм | Длина хлыста |
| `PartCount` | 1 шт. | Кол-во деталей из одного хлыста |

### 8.2 Типы плоскостей реза

**Прямой рез** (1 плоскость, нормаль ⊥ оси трубы):
- Контур = окружность; `ΔY(θ) = 0`; чистая ротация 360°

**Косой рез** (1 плоскость, нормаль под углом):
- Контур = эллипс (синусоида в развёртке)
- `ΔY(θ) = r × (nz·cos(θ) − nx·sin(θ)) / ny`

**Седловидный рез** (2 плоскости):
- Контур = `min(Y₁(θ), Y₂(θ))` — составная кривая
- Точки перехода (ребро): где `Y₁(θ) = Y₂(θ)`
- Алгоритм: вычислить Y₁(θ) и Y₂(θ); найти θ₁, θ₂; на [θ₁,θ₂] — одна плоскость, на [θ₂,θ₁+2π] — другая

**Длина контура:**
```
CutLength = ∫₀²π √((dX/dθ)² + (dY/dθ)² + (dZ/dθ)²) dθ
Прямой рез: CutLength = π × D
```

### 8.3 Контур реза (жёлтая лента)

Функция `setCutContour(yPerDeg[360], cx, cz, r, rotOffsetDeg)`:
- Массив из 360 Y-значений (по одному на каждый градус θ)
- Лента шириной 1.2 мм, приподнята на 0.3 мм (без z-fight)
- Цвет: жёлтый (1.0, 0.9, 0.0)
- Вращается вместе с трубой через ту же `tubeTransform`
- Формула для CutBefore: `contour[k] = −laserOffsetY − fY`
- Формула для CutAfter: `contour[k] = fY`

### 8.4 Функция `setTubeTransform(rotYDeg, feedY, px, py, pz)`

```
M·v = Ry(rotYDeg) × (v − pivot) + pivot + (0, −feedY, 0)
где pivot = (px, py, pz) = (−76, 0, 347)
```
- `feedY > 0` → труба сдвигается в −Y (к лазеру)
- `feedY < 0` → труба сдвигается в +Y (от лазера)

### 8.5 Фазы 10-фазной анимации

```
┌─────────────────────┐
│ LCutBeforeFeed (0)  │  ← Подача прямой трубы к лазеру (progressive geometry)
│ LCutBefore (1)      │  ← Ротация 360° + ΔY(θ), луч ON, жёлтый контур
│ LReturnFromCut (2)  │  ← Возврат трубы от лазера к станку
├─────────────────────┤
│ LFeed (3)           │  ← Подача (progressive geometry)
│ LRotation (4)       │  ← Ротация (progressive geometry)
│ LBend (5)           │  ← Гибка (progressive geometry)
│ LTubePush (6)       │  ← Проталкивание на ClampLength
│ LBenderReturn (7)   │  ← Возврат ролика
├─────────────────────┤
│ LCutAfterFeed (8)   │  ← Подача согнутой трубы к лазеру
│ LCutAfter (9)       │  ← Ротация 360° + ΔY(θ), луч ON, жёлтый контур
└─────────────────────┘
```

Фазы 3–7 повторяются для каждого `step[i]` программы гибки.

**Фаза 0: LCutBeforeFeed** — Подача к лазеру
- `feedDist = laserOffsetY + maxDeltaCutBefore ≈ 61.07 мм` (для тестовой модели)
- Progressive geometry: красный кончик растёт в −Y, синий хвост укорачивается
- `currentFeed = feedDist × progress`
- `setTubeTransform` не используется; луч OFF

**Фаза 1: LCutBefore** — Рез контура
- `rotAngle = progress × 360`
- `theta = rotAngle × π / 180`
- `feedDelta = cutBefore.contourY(theta, r) − offset`
- `setTubeTransform(rotAngle, −feedDelta, PT0_X, PT0_Y, PT0_Z)`
- Луч ON; жёлтый контур виден

**Фаза 2: LReturnFromCut** — Возврат от лазера
- Меш пересобирается с cutBefore-контуром (видны седловидный торец)
- `currentFeed = feedBase × (1 − progress)`
- Луч OFF

**Переход:** `clearTubeTransform()` → `rebuildLaserBendGeometry()` (с `alreadyFed = cutBeforeFeedDist`)

**Фазы 3–7 (гибочный цикл):** идентичны основной анимации гибки. Progressive geometry без `setTubeTransform`. Синий хвост (хлыст) остаётся видимым.

**Фаза 8: LCutAfterFeed** — Подача согнутой трубы к лазеру
- Прямой сегмент подаётся до лазера после всех шагов гибки

**Фаза 9: LCutAfter** — Финальный рез
- Аналогично фазе 1, но для `cutAfter` (прямой рез → `ΔY = 0`)
- После 360° луч OFF; красный контур остаётся на трубе

**Серийное производство:** при `PartCount > 1` — цикл повторяется. Расчёт максимума из хлыста: `floor(WhipLength / LTotal)`.

## 9. Импорт STEP-модели

### 9.1 Алгоритм распознавания трубы

1. `CYLINDRICAL_SURFACE` → пары наружный/внутренний → `D`, толщина стенки
2. `TOROIDAL_SURFACE` → `R`, угол гиба (`arccos(d₁·d₂)`)
3. Длины прямых участков по расстоянию между торцами
4. На каждом торце найти `PLANE`-поверхности:
   - Нормаль ⊥ оси → прямой рез
   - Нормаль наклонена → косой рез
   - Два наклонных `PLANE` → седловидный рез

### 9.2 Преобразование нормалей в локальную СК

- `tubeY` направлен от торуса (соединённый конец) к свободному концу
- `tubeX = cross(tubeY, refUp)`, `tubeZ = cross(tubeX, tubeY)`
- Нормаль плоскости проецируется на `(tubeX, tubeY, tubeZ)` → локальные `(nx, ny, nz)`

### 9.3 Экспорт STEP AP242

```cpp
BRepPrimAPI_MakeCylinder(gp_Ax2(...), r, length)
BRepPrimAPI_MakeTorus(gp_Ax2(...), R, r_tube, angle_rad)
STEPControl_Writer writer;
writer.Transfer(shape, STEPControl_AsIs);
writer.Write("output.stp");
```

## 12. STEP_DONE: синхронизация EXE ↔ реальный станок

### 12.1 Контур управления (целевой — с Teensy)

```
EXE (Master программы)
  │ Ethernet HTTP/JSON
  ▼
Teensy 4.1 (Motion Controller + Modbus Slave)
  │ RS-485 Modbus RTU
  ▼
PLC110 (безопасность, гибочная ось, педали, interlock)
  │
  ├─ Релейные DO (гибка, вспомогательные цепи)
  └─ MR-JE Z/C через Line Driver Step/Dir
```

### 12.2 Принцип синхронизации

1. EXE отправляет один шаг: `{step_index, feed, rotate, bend, uuid}`
2. Teensy/ESP переводит систему в `BUSY`, передаёт команду
3. По завершении физического шага формирует `STEP_DONE(step_index, actual_pos, actual_time, alarms)`
4. EXE продвигает симуляцию **только после `STEP_DONE`** с совпадающими `step_index` и `uuid`
5. При рассогласовании (`timeout`, `alarm`, `unexpected step`) — пауза, фиксация ошибки, ручное подтверждение

### 12.3 Минимальный набор статусов

| Поле | Смысл |
|------|-------|
| `machine_state` | `IDLE / ARMED / BUSY / HOLD / ALARM` |
| `program_id` | Идентификатор загруженной программы |
| `step_index` | Текущий шаг в железе |
| `ack_index` | Последний подтверждённый шаг |
| `z_actual`, `c_actual`, `bend_actual` | Фактические координаты/угол |
| `alarm_code` | Код аварии привода/ПЛК |

### 12.4 Защита от рассинхронизации

- Команды EXE идемпотентны: `program_id + step_index + crc`
- Каждый шаг подтверждается только один раз (`ACK once`)
- После обрыва связи: `HOLD`, продолжение только после `RESYNC`
- `RESYNC`: сверка фактических координат с ожидаемыми + допуски

### 12.5 Текущий этап (без Teensy, ESP8266)

На текущем этапе (ESP8266 как шлюз):
- Оси Z и C работают в режиме **Dry Run (виртуально)** — EXE симулирует завершение шагов
- Физическое движение выполняется **только для оси гибки (ПЛК)**
- Полное реальное движение Z/C переносится на этап Teensy

## 14. Критерии приёмки

### 14.1 Гибка (FR тз.md)

1. ≥50 шагов без ошибок интерфейса; корректный `L_total`
2. Цвета зон синий/красный/зелёный корректны для каждого шага
3. Шаг 90°, R=76: `L_bend ≈ 119.38 мм`
4. Дуги плавные без заметных швов
5. Экспорт/импорт JSON восстанавливает те же шаги и `L_total`
6. Клик по сегменту в 3D активирует нужную строку в таблице
7. STL станка полупрозрачен; ориентация совпадает с системой координат
8. Контрольный сценарий `циклы гиба1.png` воспроизводится по цветам и кинематике
9. При подаче сформированная часть (красный+зелёный) сдвигается в −Y
10. Валидация диапазонов: значения вне ограничений блокируются с ошибкой
11. Фаза гибки: `v_feed = R × ω_bend` в пределах анимационного шага
12. D и R едины для всей программы; смешение невозможно
13. Анимация: 7 фаз (A-B-B2-C-D-E-F) строго последовательно; подсветка фазы
14. Направление гиба — в +X; трубогиб вращается против часовой (при виде с камеры)

### 14.2 Лазерная резка (FR тз_v2.0)

1. При запуске лазерной анимации видна синяя труба (хлыст = WhipLength мм)
2. Жёлтый контур реза виден на поверхности трубы в фазах LCutBefore/LCutAfter
3. Текущий шаг подсвечен зелёным в таблице во время анимации
4. Прогресс-бар с названием фазы виден всегда
5. При PartCount > 1 анимация циклически повторяется
6. Плавный возврат после CutAfter (без телепортации)
7. Пресеты оснастки: при выборе D/R/ClampLength/LaserOffsetY обновляются автоматически
8. Масштабирование UI: Ctrl+/- работает; режим «Тачскрин» увеличивает элементы
