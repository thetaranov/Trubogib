## 3. Три оси станка

### 3.1. Ось гибки (РЕАЛИЗОВАНА — управляется ПЛК)

| Параметр | Значение |
|----------|----------|
| Привод | Мотор + редуктор на гибочном ролике |
| Обратная связь | Энкодер 360 P/R (A/B квадратура) |
| Управление | Owen PLC110-30 через DO10/DO11 |
| Режим | Ручной (педали) / Автоматический (по углу) |

**Ключевые входы/выходы ПЛК:**
- Быстрые входы: `%IX0.0` (энк. A), `%IX0.1` (энк. B)
- Дискретные входы: `%IX1.4` D5 (режим), `%IX1.5` D7 (педаль вперёд), `%IX1.6` D8 (педаль назад)
- Дискретные выходы: `%QX3.0.2` DO7 (SON Z), `%QX3.0.3` DO8 (SON C), `%QX3.0.5` DO10 (мотор+), `%QX3.0.6` DO11 (мотор−), `%QX5.0` (бипер)

### 3.2. Ось Z — подача каретки (В РАЗРАБОТКЕ — ожидает Teensy)

| Параметр | Значение |
|----------|----------|
| Серводвигатель | HG-SN152J-S100 (1.5 кВт) |
| Серводрайвер | MR-JE-200A |
| Редуктор | PF115L1-008 (i = 1:8, KПД ≈ 95%) |
| Шестерня | Прямозубая M2, **Z35** (делительный d = 70 мм) |
| Рейка | Прямозубая M2, L = 6000 мм |
| Направляющие | HGR20 × 2 шт + HGH20CA × 4 шт |
| Ход | 0–6000 мм |
| Скорость подачи (рабочая) | до 300 мм/с |

**Кинематический расчёт (ось Z):**
```
Шестерня M2 Z35:
  d_делительный  = m × Z = 2 × 35 = 70 мм
  Длина окружности = π × 70 = 219.91 мм

Передаточное число редуктора: i = 8
Ход на 1 оборот вала двигателя: 219.91 / 8 = 27.49 мм
Максимальная скорость подачи (2000 об/мин двиг.): 27.49 × 2000 / 60 = 916.3 мм/с
Усилие подачи: 7.16 × 8 × 0.95 / (0.070/2) ≈ 1555 Н

Энкодер двигателя: 262144 ppr (18 бит)
Разрешение позиционирования: 219.91 / (262144 × 8) ≈ 0.000105 мм/импульс
```

**Посадки и крепления (ось Z):**
- Модуль всех шестерён системы: **m = 2**
- Шестерня Z35 (пиньон): посадочное отверстие **ø25** (при необходимости с шпонкой — ø28)
- Шестерня Z105 (на патроне): крепится через трубную втулку **ø54**; патрон опирается на 2 подшипника **6211-2RS**

**⚠️ Анализ люфта Z35:**
- Шестерня Z35, m2: df=65 мм, вал ø25 → стенка 20 мм ✅ (z20: стенка 5 мм — разрушится)
- Зацепление с рейкой даёт ~0.05–0.1 мм люфт без специальных мер
- Для лазерной резки рекомендуется: **подпружиненная разрезная шестерня** или косозубая пара

### 3.3. Ось C — вращение патрона (В РАЗРАБОТКЕ — ожидает Teensy)

| Параметр | Значение |
|----------|----------|
| Серводвигатель | HG-SN152J-S100 (1.5 кВт) |
| Серводрайвер | MR-JE-200A |
| Редуктор | PF115L2-020 (i = 1:20, КПД ≈ 92%) |
| Шестерня малая (пиньон) | M2, **Z35** (d_дел = 70 мм, df = 65 мм) |
| Шестерня большая (на патроне) | M2, **Z105** (da = 214 мм, df = 205 мм) |
| Межосевое расстояние | **140 мм** |
| Патрон | токарный 200 мм, Р3700 (7100-0007В ГОСТ 2675-80) |
| Диапазон | непрерывное вращение 360°+ (в обе стороны) |
| Скорость | до 120°/с |
| Точность | ±0.1° |

**Кинематический расчёт (ось C):**
```
Передаточное планетарного редуктора: i₁ = 20
Передаточное зубчатой пары Z35/Z105: i₂ = 105/35 = 3
Суммарное передаточное: i_total = 20 × 3 = 60:1

1 оборот патрона = 60 оборотов двигателя
Скорость вращения патрона (2000 об/мин двиг.): 2000 / 60 ≈ 33.33 об/мин
Момент на патроне: 7.16 × 20 × 0.92 = 131.7 Н·м
Разрешение: 360° / (262144 × 60) ≈ 0.0000229°/импульс

Почему Z35/Z105, а не Z20/Z60:
  z20 m2: df=35мм, вал ø25 → стенка всего 5мм — разрушится при нагрузке
  z35 m2: df=65мм, вал ø25 → стенка 20мм ✅
  z105 da=214мм > ø200мм патрона → большая шестерня служит фланцем патрона ✅
```

**Посадка Z105 на патрон Р3700:**
- Центрирование: буртик **ø165 H7** проточен на шестерне
- Крепление: **3× отв. ø11 (М10)** на PCD **174 мм**, шаг 120°
- Центральное отверстие: **ø50 мм** (сквозное через шестерню и вал)
- Биение после установки: ≤ 0.03 мм (требование лазерной резки)

**⚠️ Анализ люфта оси C:**

| Метод снижения люфта | Эффект | Сложность |
|---------------------|--------|-----------|
| Подпружиненная разрезная шестерня Z105 | Убирает люфт зубчатой пары | Средняя |
| Планетарный редуктор (вместо планетарного + открытой пары) | Люфт 1-3' | Дорого |
| Ременная передача GT2/GT3 | Минимальный люфт | Просто |

- Суммарный люфт схемы «планетарный + открытая пара»: до **10 угловых минут**
- При R=100 мм (труба ø200): люфт 10' → **0.29 мм** погрешность (критично для лазерной резки)

## 6. Управляющая программа (формат)

### Формат шага

| Поле | Тип | Единица | Описание |
|------|-----|---------|----------|
| `L_feed` | float | мм | Подача каретки (ось Z) |
| `Rotation` | float | ° | Вращение патрона (ось C) |
| `Angle` | float | ° | Угол гибки |

### Глобальные параметры программы

| Параметр | Описание |
|----------|----------|
| `D` | Наружный диаметр трубы (мм), единый для всех шагов |
| `R` | Радиус гиба по средней линии (CLR, мм), единый для всех шагов |
| `ClampLength` | Длина зажима ролика (мм), по умолч. 60 |

### Расчётные формулы

```
Длина дуги на шаге:     L_bend = π × R × Angle / 180
Полная длина заготовки:  L_total = Σ(L_feed) + Σ(L_bend)
Инвариант:              L_blue + L_red + L_green = L_total (на любом кадре анимации)
```

### Ограничения ввода

```
D > 0,  R > 0,  ClampLength ≥ 0
0 ≤ Angle ≤ 180°
Rotation ∈ [−360; +360]° на шаг
L_feed ≥ ClampLength (если два соседних шага имеют Angle > 0)
```

### Последовательность выполнения одного шага

```
a) ПОДАЧА (Feed):      Teensy → MR-JE-200A #1 → каретка на L_feed мм
b) РОТАЦИЯ (Rotation): Teensy → MR-JE-200A #2 → патрон на Rotation°
c) ГИБКА (Bend):       Teensy → ПЛК (Modbus) → гибка на Angle°
d) ОТВОД (Clearance):  каретка +ClampLength мм (освобождение ролика)
e) ВОЗВРАТ РОЛИКА:     ПЛК → мотор назад до 0°
f) ВОЗВРАТ КАРЕТКИ:    каретка −ClampLength мм (назад в рабочую позицию)
```

## 8. Дорожная карта

### Этап 1 — ТЕКУЩИЙ: ПЛК + ESP8266 (полуавтоматический гиб)

```
Статус: В РАЗРАБОТКЕ

Возможности:
  [✓] Ось гибки — автоматическая (ПЛК + энкодер + мотор)
  [ ] Ось Z (подача) — ручная (серводрайвер УСТАНОВЛЕН, Step/Dir НЕ управляется)
  [ ] Ось C (вращение) — ручная (серводрайвер УСТАНОВЛЕН, Step/Dir НЕ управляется)

Ключевое ограничение:
  ESP8266 НЕ способен генерировать Step/Dir (нет нужных аппаратных таймеров).
  Для осей Z и C требуется Teensy 4.1.

Контур управления:
  EXE v2.1 ──HTTP/REST──> ESP8266 ──RS-485 Modbus──> ПЛК ──DO10/DO11──> Ось гибки

Задачи:
  1. Доработать ESP8266: REST API + расширенные Modbus-регистры
  2. Доработать ПЛК: интеграционный автомат + экспорт статуса
  3. Доработать EXE: Machine Runtime + панель управления
  4. Dry Run для осей Z/C (виртуальное выполнение в EXE, физически — ожидание)
  5. Интеграционное тестирование EXE → ESP → ПЛК → мотор
```

### Этап 2 — БУДУЩИЙ: Teensy 4.1 (полный ЧПУ)

```
Статус: МАКЕТНАЯ ПЛАТА СОБРАНА, ГОТОВА К ПОДКЛЮЧЕНИЮ

Изменения в архитектуре:
  - Teensy ЗАМЕНЯЕТ ESP8266 (Ethernet вместо WiFi)
  - Teensy генерирует Step/Dir для осей Z и C
  - ПЛК остаётся для оси гибки
  - REST API переносится из ESP в Teensy без изменений
  - Карта Modbus-регистров расширяется (существующие адреса не меняются)

Контур управления:
  EXE ──Ethernet──> Teensy 4.1
                      ├── Step/Dir ──> Серво Z (подача)
                      ├── Step/Dir ──> Серво C (вращение)
                      └── Modbus RS-485 ──> ПЛК ──> Ось гибки

Возможности:
  [✓] Ось гибки — автоматическая (ПЛК)
  [✓] Ось Z (подача) — автоматическая (Teensy + серво)
  [✓] Ось C (вращение) — автоматическая (Teensy + серво)
  [✓] Полный автоматический цикл гибки
```

### Принцип обратной совместимости

- **EXE** меняет только транспортный слой (HTTP/WiFi → Ethernet), протокол команд остаётся
- **ПЛК** не меняется — продолжает управлять осью гибки
- **Карта Modbus** расширяется, но существующие адреса сохраняют значения
