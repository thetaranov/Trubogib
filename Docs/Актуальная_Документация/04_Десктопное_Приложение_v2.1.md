## 2. Архитектура UI и Окон

### 2.1 Принцип FBO + ImGui Docking

```
1. 3D-сцена рендерится в FBO (Framebuffer Object) — off-screen текстура
2. Текстура FBO отображается внутри ImGui-окна через ImGui::Image()
3. 3D-контент строго ограничен рамками своего окна
4. Все окна управляются через ImGui Docking — единая система привязки
```

**Pipeline рендеринга (порядок вызовов каждый кадр):**
```
1. app.update()          — логика, анимация, хэш программы
2. app.renderToFBO()     — 3D → FBO (glBindFramebuffer → render → unbind)
3. glClear(screen)       — очистка экрана
4. app.drawUI()          — ImGui (включая Image() с FBO текстурой)
5. ImGui::Render()       — финализация
6. glfwSwapBuffers()
```

FBO пересоздаётся (RGBA8 + DEPTH24_STENCIL8), если размер content-области окна «Сцена» изменился.

## 3. Геометрическое ядро

### 3.1 Система координат и Point0

| Ось | Направление |
|-----|-------------|
| +Y | От камеры к станку |
| −Y | Направление подачи (труба выезжает к камере) |
| +X | Вправо на экране (направление гиба) |
| Z | Вертикальная ось сцены |

```cpp
// Из app.cpp (строки 24–26) — константы Point0:
static constexpr float PT0_X = -76.0f;
static constexpr float PT0_Y =   0.0f;
static constexpr float PT0_Z = 347.0f;
```

**Point0 = (−76, 0, 347) мм** — фиксированная точка начала пластической деформации.

Начальная камера: yaw = −90°, pitch = 25°, dist = 800, center ≈ (−76, −100, 347).

### 3.2 Термины и обозначения

| Термин | Описание |
|--------|----------|
| **D** | Наружный диаметр трубы (мм), глобальный для всей программы |
| **R (CLR)** | Center Line Radius — радиус гиба по средней линии (мм) |
| **ClampLength** | Длина прижима (мм), по умолчанию 60 мм |
| **StraightLen** | Минимальный прямой участок между соседними гибами (ограничение оснастки) |
| **L_feed** | Подача — линейное перемещение заготовки (мм) |
| **Rotation** | Поворот вокруг продольной оси (°), правило правой руки |
| **Angle** | Угол гиба (°) |
| **L_bend** | `π × R × Angle / 180` |
| **L_total** | `Σ(L_feed_i) + Σ(L_bend_i)` |
| **WhipLength** | Длина хлыста (по умолчанию 6000 мм) |
| **PartCount** | Количество деталей из хлыста |

### 3.3 Цветовое кодирование зон трубы

| Цвет | Зона |
|------|------|
| **Синий** | Материал до Point0 (хлыст, необработанный) |
| **Красный** | Прямые участки после Point0 |
| **Зелёный** | Дуги (гибы) после Point0 |

**Инвариант:** `L_синий + L_красный + L_зелёный = L_total` на каждом кадре.

Хлыст визуализируется от Point0 в **+Y**: `blueLen = max(0, fullLTotal − consumed)`.

### 3.4 Алгоритм обратного обхода (`buildTubeGeometry`)

Геометрия строится **в обратном порядке шагов** (от N−1 к 0), чтобы корректно воспроизвести физику проталкивания:

```
Начальные условия:
    pos   = Point0 = (−76, 0, 347)
    fwd   = (0, −1, 0)    // −Y: направление подачи
    up    = (0,  0, 1)    // Z вверх
    right = cross(up, fwd) = (+1, 0, 0)  // +X: направление гиба

for i от N−1 до 0:
    // 1. Гиб (ВСЕГДА в XY — ролик фиксирован, ДО ротации)
    if step[i].Angle > 0:
        createArc(pos, fwd, up, right, R, radius, Angle_i)
        обновить pos / fwd / up (right меняется через rotateAround)

    // 2. Ротация (твист фрейма, влияет на последующие сегменты)
    if step[i].Rotation != 0:
        up    = rotateAround(up,    fwd, Rotation_i)
        right = rotateAround(right, fwd, Rotation_i)

    // 3. Подача (прямой красный сегмент)
    if step[i].L_feed > 0:
        createCylinder(pos, pos + fwd * L_feed, radius, Red)
        pos += fwd * L_feed

// Синий хвост от Point0 в +Y
blueLen = max(0, fullLTotal − consumed)
createCylinder(Point0, Point0 + (0,+1,0)*blueLen, radius, Blue)
```

**Почему гиб ДО ротации в обратном обходе:** ролик физически фиксирован в XY. В прямом порядке — сначала ротация, потом гиб. В обратном — порядок переставляется.

### 3.5 Сечение (`createCylinder`, `createArc`)

```cpp
// tube_geometry.cpp: фрейм поперечного сечения
V3 refUp = (abs(dir.z) < 0.9) ? V3(0,0,1) : V3(1,0,0);
V3 tubeX = refUp.cross(dir).norm();
V3 tubeZ = dir.cross(tubeX).norm();

// Кольцо точек:
n(θ) = tubeZ * cos(θ) + tubeX * sin(θ)
p(θ) = center + n(θ) * radius

// θ=0 → верхняя образующая (максимум Z для трубы с dir=(0,−1,0))
```

Количество сегментов сечения: `CIRC_SEGMENTS = 32`.
Шаг дуги: 1 кольцо на 5°, минимум 4 кольца (`arcSteps = max(4, angleDeg / 5)`).

### 3.6 V-cut якоря (плоскости реза)

При построении торца с резом (`CutDefinition`):
```
minDeltaStart = min по θ: (contourY(θ, r) - offset0)
maxDeltaEnd   = max по θ: (contourY(θ, r) - offset0)

// Каждая вершина торца смещается вдоль dir:
p += dir * (fY(θ) - min_or_maxDelta)
```
Это обеспечивает совмещение контура реза с реальным торцом трубы.

### 3.7 Контрольный пример (D=38, R=76)

Программа: Шаг1 — L=100, A=90°; Шаг2 — L=200, A=0°

```
L_total = 100 + π×76×90/180 + 200 ≈ 419.38 мм

Red2 (шаг 2): от (−76, 0, 347) до (−76, −200, 347)   — в −Y, 200 мм
Green1 (шаг 1): дуга R=76 от (−76, −200, 347),
                центр = (0, −200, 347), угол 90°,
                конец ≈ (0, −276, 347)
Red1 (шаг 1):  от (0, −276, 347) в +X, 100 мм → (100, −276, 347)
Синий хвост:   от (−76, 0, 347) в +Y, 0 мм (весь материал потреблён)
```

### 4.2 Гибка — 7 фаз на шаг (сверено с `app.h:83`)

```cpp
// app.h, строка 83:
enum AnimPhase {
    AnimFeed=0, AnimRotation=1, AnimClampClose=2, AnimBend=3,
    AnimClampOpen=4, AnimTubePush=5, AnimBenderReturn=6
};
```

| # | Фаза | Скорость | Прижим | STL ролика |
|---|------|----------|--------|------------|
| 0 | **Feed** — подача | 300 мм/с | Открыт | Неподвижен (0°) |
| 1 | **Rotation** — ротация | 120°/с | Открыт | Неподвижен (0°) |
| 2 | **ClampClose** — зажим | 600 мм/с, 150 мм | Закрывается | Неподвижен (0°) |
| 3 | **Bend** — гибка | 60°/с | Зажат | 0° → Angle |
| 4 | **ClampOpen** — отвод прижима | 600 мм/с, 150 мм | Открывается | Неподвижен на Angle |
| 5 | **TubePush** — подача на clearance | 300 мм/с, ClampLength мм | Открыт | Неподвижен на Angle |
| 6 | **BenderReturn** — возврат ролика | 120°/с | Открыт | Angle → 0° |

**Правило пропуска:** если `step.bendAngle < 0.01`, фазы 2–6 (ClampClose → BenderReturn) пропускаются:
```cpp
// app.cpp ~строка 504:
if (step.bendAngle < 0.01 && m_animPhase == AnimClampClose)
    m_animPhase = AnimBenderReturn + 1; // skip to next step
```

**Соответствие benderAngle:**
```
Feed / Rotation / ClampClose : benderAngle = 0
Bend                         : benderAngle = step.Angle * progress
ClampOpen / TubePush         : benderAngle = step.Angle
BenderReturn                 : benderAngle = step.Angle * (1 − progress)
```

**Порядок D→E строго обязателен:** ClampOpen (фаза 4) ДОЛЖЕН отработать до TubePush (фаза 5).

### 4.5 Позиционирование лазерной головки

```
LaserPoint.X = PT0_X = −76 мм
LaserPoint.Y = −LaserOffsetY = −60 мм (по умолч.)
```

### 4.6 `setTubeTransform` — матрица движения трубы при резе

```
M·v = Ry(rotYDeg) × (v − pivot) + pivot + (0, −feedY, 0)
где pivot = (−76, 0, 347)
feedY > 0 → труба смещается в −Y (к лазеру)
```

Используется только в фазах реза (LCutBefore, LCutAfter). После `LReturnFromCut` — `clearTubeTransform()`.

### 4.7 Контур реза (жёлтая лента)

Функция `setCutContour(yPerDeg[360], cx, cz, r, rotOffsetDeg)`:
- 360 Y-значений (по одному на каждый градус θ).
- Лента шириной 1.2 мм, приподнята на 0.3 мм (без z-fighting).
- Цвет: жёлтый `(1.0, 0.9, 0.0)`.
- Вращается вместе с трубой через ту же `tubeTransform`.

**Формулы сечения трубы (для красных сегментов с dir=(0,−1,0)):**
```cpp
// renderer.cpp (подтверждено критерий_приемки_резки.md):
x  = cx + r * sinf(th);   // sin(θ) → X
z  = cz + r * cosf(th);   // cos(θ) → Z
nx = sinf(th);             // нормаль по X
nz = cosf(th);             // нормаль по Z
// θ=0 → верхняя образующая (Z_max)
```

**Типы плоскостей реза:**

| Тип | Плоскостей | Контур | ΔY(θ) |
|-----|-----------|--------|-------|
| Прямой | 1 (нормаль ⊥ оси) | Окружность | 0 |
| Косой | 1 (нормаль под углом) | Эллипс/синусоида | `r·(nz·cos(θ) − nx·sin(θ)) / ny` |
| Седловидный | 2 | `min(Y₁(θ), Y₂(θ))` | Составная кривая |

### 4.8 Серийное производство

- Параметры: `WhipLength`, `PartCount`.
- После завершения детали запускается следующая, пока хватает хлыста и не достигнут `PartCount`.
- Хлыст в начале = `whipLength`; после каждой детали: `whipRemaining -= lTotal`.
- Максимум из хлыста: `floor(whipLength / lTotal)`.
- Прогресс по деталям отображается в UI.

## 5. Система контроля столкновений (Logic2)

### 5.1 Два режима (из `ui.useLogic2`)

| Режим | Функция | Меши | Алгоритм |
|-------|---------|------|----------|
| `useLogic2 = false` | `checkCollisions()` | STL модели станка | Точечная проверка centerline |
| `useLogic2 = true` | `checkCollisionsV2()` | STL бендер + ролик + прижим + пол | Capsule-triangle + AABB |

### 5.2 Статическая проверка (checkCollisionsV2)

**Шаг 1: pre-check прямого участка между соседними гибами**
```cpp
// app.cpp ~строка 1980:
if (prev.bendAngle > 0 && cur.bendAngle > 0)
    if (cur.feedLength < StraightLen)
        → ERROR: "прямой участок %.1f мм < мин. %.1f мм"
```

**Шаг 2: построение centerline (в обратном порядке, совпадает с tube_geometry)**

Точки centerline: начало от Point0, затем дуги по 10 равных шагов каждая, затем прямые.

**Шаг 3: проверка каждой точки на зоны исключения**

Три зоны исключения (из `app.cpp` строка 1947):
```cpp
double exclusionRadius = D * 1.5 / 2.0;  // 1.5 × радиус трубы
```

| Зона | Описание |
|------|---------|
| 1. Цилиндр подачи | XZ-расстояние от Point0 < exclR, Y ∈ [0, R] |
| 2. Торусная зона | Расстояние от точки дуги < exclR (θ от 0 до lastBendAngle) |
| 3. Цилиндр прижима | Вдоль направления касательной дуги, длина = clampLen + tubeRadius |

Точка в зоне исключения → **пропустить** (не проверять на коллизию).

**Шаг 4: проверка против STL-мешей**
- Пол (floor): `p.z < m_floorZ + tubeRadius`.
- STL меши: AABB-глобальный rejection → AABB-на-треугольник → точечная проверка.

### 5.3 Realtime коллизия (во время анимации, Logic2)

Функция `checkAnimCollision()` (capsule-triangle):

```
Алгоритм (segTriDist2, app.cpp строка 2236):
1. Сегмент центральной линии (s0, s1) + radius трубы = капсула
2. AABB rejection для сегмента: segment_AABB + radius
3. Per-mesh AABB rejection
4. Per-triangle AABB rejection
5. segTriDist2(s0, s1, t0, t1, t2) < radius²  → коллизия

Преимущества capsule vs точечная:
- нет слепой зоны между sample-точками
- детект ближе к реальному контакту
```

**Правило исключения для realtime (из тз.md):**
- Зона исключения расширена до `1.5 × D / 2`.
- Сегмент исключается, если **хотя бы один его конец** попадает в зону исключения.

При обнаружении коллизии — пауза анимации с popup.

### 5.4 Зона исключения для визуализации (в update())

```cpp
// app.cpp строка 121 — для отображения зоны в 3D:
double exclR = D * 1.2 / 2.0;
```
Заметка: для визуализации используется `1.2×`, для расчёта коллизий — `1.5×`.

### 6.3 Импорт STEP (OCCT)

**Алгоритм распознавания трубы:**
1. `CYLINDRICAL_SURFACE` → D, толщина стенки
2. `TOROIDAL_SURFACE` → R, угол гиба `(arccos(d₁·d₂))`
3. Прямые участки — по расстоянию между торцами
4. На каждом торце: найти `PLANE`-поверхности → тип реза (прямой/косой/седловидный)

**Преобразование нормалей:**
- `tubeY` — от торуса к свободному концу
- `tubeX = cross(tubeY, refUp)`, `tubeZ = cross(tubeX, tubeY)`
- Нормаль проецируется на `(tubeX, tubeY, tubeZ)` → локальные `(nx, ny, nz)`

При несоответствии D/R из файла текущей оснастке → popup переоснастки.

## 8. Синхронизация EXE ↔ Станок (STEP_DONE)

### 8.1 Контур управления (целевой — с Teensy)

```
EXE (Master)
  │ Ethernet HTTP/JSON
  ▼
Teensy 4.1 (Motion Controller + Modbus Slave)
  │ RS-485 Modbus RTU
  ▼
PLC110 (безопасность, гибочная ось, педали, interlock)
  │
  ├─ Релейные DO → ВЧП (гибочная ось)
  └─ MR-JE Z/C через Line Driver Step/Dir
```

### 8.2 Принцип синхронизации

1. EXE отправляет один шаг: `{step_index, feed, rotate, bend, uuid}`
2. Teensy/ESP → `BUSY`, передаёт команду на железо
3. По завершении физического шага → `STEP_DONE(step_index, actual_pos, actual_time, alarms)`
4. EXE продвигает симуляцию **только после `STEP_DONE`** с совпадающими `step_index` + `uuid`
5. При рассогласовании (timeout, alarm, unexpected step) → пауза, фиксация ошибки, ручное подтверждение

### 8.3 Статус машины

| Поле | Смысл |
|------|-------|
| `machine_state` | `IDLE / ARMED / BUSY / HOLD / ALARM` |
| `step_index` | Текущий шаг в железе |
| `ack_index` | Последний подтверждённый шаг |
| `z_actual`, `c_actual`, `bend_actual` | Фактические координаты/угол |
| `alarm_code` | Код аварии |

### 8.4 Текущий этап (ESP8266, без Teensy)

- Оси Z и C работают в **Dry Run** — EXE симулирует завершение шагов.
- Физическое движение — **только ось гибки (ПЛК)**.
- Полное реальное движение Z/C переносится на этап Teensy.
