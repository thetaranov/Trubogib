2.2. Схема подключения энкодера к ПЛК

Энкодер BRT38                Owen PLC110-30
─────────────                ─────────────
+V  (красный)  ────────────► +24В (от БП ПЛК)
0V  (чёрный)   ────────────► 0В GND
A   (зелёный)  ────────────► Быстрый вход %IX0.0  (HSC канал A)
B   (белый)    ────────────► Быстрый вход %IX0.1  (HSC канал B)
Экран          ────────────► Клемма PE (только с одной стороны)
Быстрые входы ≠ обычные входы! Энкодер подключается только к клеммам БЫСТРЫХ входов (%IX0.0 и %IX0.1). Обычные дискретные входы не способны считать импульсы на скорости 1000+ Гц. В CoDeSys: PLC Configuration → Fast Inputs → Channel 0 → Quadrature A/B.

2.3. Расчёт угла гибки

Энкодер 360 P/R × 4x квадратура = 1440 отсчётов на оборот вала

В текущей прошивке ПЛК применено упрощение:
  gCurrentAngle = HSC_value
  (1 импульс ≈ 1 градус — работает при передаточном числе механики ~4:1)

Если передаточное число другое → пересчитать:
  Угол (°) = HSC_value × 360 / (360 × 4 × i_механики)

5. Алгоритм работы (PLC_PRG)

Каждый цикл ПЛК (~10 мс):

[1] Чтение входов:
    gMode         ← %IX1.4
    pedalFwd      ← %IX1.5
    pedalRev      ← %IX1.6
    gCurrentAngle ← HSC_value

[2] Обработка Modbus-команд от ESP/Teensy:
    IF gHostCommand = 1 → HSC_value := 0; gCurrentAngle := 0; gHostCommand := 0
    IF gWebTarget > 0  → gTargetAngle := gWebTarget; gWebTarget := 0

[3] Ручной режим (gMode = FALSE):
    pedalFwd → DO10 (мотор+)
    pedalRev → DO11 (мотор−)
    Блокировка: DO10 и DO11 никогда не активны одновременно

[4] Автоматический режим (gMode = TRUE):
    Цикл 1 (гибка вперёд):
      Педаль D7 управляет DO10
      IF gCurrentAngle >= gTargetAngle → gDone:=TRUE, beeper 500мс, gCycleNum:=2

    Цикл 2 (возврат):
      Педаль D8 управляет DO11
      IF gCurrentAngle <= 0 → gDone:=TRUE, beeper 500мс, gCycleNum:=1

[5] Запись Modbus-регистров (FC 0x10, каждые 100 мс):
    HR[0]←gCurrentAngle, HR[1]←gTargetAngle, HR[2]←gCycleNum,
    HR[3]←gMode, HR[4]←gDone, HR[5]←pedalFwd, HR[6]←pedalRev

[6] Чтение Modbus-регистров (FC 0x03, каждые 100 мс):
    HR[9]→gHostCommand, HR[10]→gTargetAngle

8. Загрузка программы в ПЛК — пошаговая инструкция

  1. Установить CoDeSys v2.3.9 (с сайта овен.рф). Скопировать папку PLC110.30_m_v2/ в директорию таргетов CoDeSys. Запустить InstallTarget.bat.
  2. Файл → Новый → Target: PLC110-30 [M02] → Язык: Structured Text (ST).
  3. Проект → Импорт → выбрать ПЛК/TRUBOGIB_CLEAN.exp.
  4. ВАЖНО (двухпанельный редактор): Строки VAR...END_VAR вставить в верхнюю панель, код программы — в нижнюю панель.
  5. PLC Configuration → Fast Inputs → Channel 0: Mode = Quadrature A/B (4x). Привязать %IX0.0, %IX0.1.
  6. Discrete Inputs → привязать %IX1.4, %IX1.5, %IX1.6.
  7. Discrete Outputs → привязать %QX3.0.0, %QX3.0.1, %QX3.0.2.
  8. Проект → Перестроить всё (Ctrl+F11). Устранить все ошибки.
  9. Подключить USB (или Ethernet). Онлайн → Параметры связи → Выбрать порт.
  10. Онлайн → Загрузить. Онлайн → Старт (F5). Убедиться в зелёном индикаторе работы.

9. Калибровка (обнуление угла)

Перед первым гибом ролик должен стоять в 0°. Затем — обнулить счётчик энкодера:

  • Через десктопное приложение (EXE): кнопка «ОБНУЛИТЬ».
  • Из CoDeSys (Watch): в окне Watch добавить gCurrentAngle, вручную установить 0.
