04. Интеграционный слой

Teensy 4.1 + 4× MAX490 + 1× MAX485. Modbus RTU / USB-Serial.
Главная › 04. Интеграционный слой

1. Обзор архитектуры

Управление станком происходит либо в Ручном режиме (физические кнопки/педали на ПЛК, прямой пуск), либо в Автоматическом режиме через EXE-программу TubeBender для Windows.

В автоматическом режиме EXE обменивается данными с микроконтроллером Teensy 4.1 через USB-Serial интерфейс, используя Modbus-подобный протокол. Teensy 4.1 выполняет две задачи:

  • Step/Dir генератор: Отправляет высокочастотные импульсы (до 150 кГц) через MAX490 (линейный драйвер RS-422) по обычной витой паре (LAN кабель) к сервоприводам MR-JE-200A.
  • Modbus Slave (ID=1): Хранит регистры команд, которые ПЛК Owen PLC110-30 (Master) опрашивает через RS-485 (MAX485).

4. Modbus-команды для ПЛК (Регистры)

ПЛК Owen PLC110-30 выступает Modbus Master, Teensy — Slave (ID=1). Обмен каждые 100 мс.

4.3. Команды CMD через регистр 9 (EXE → Teensy → ПЛК)

CMDИмяДействие в ПЛК
1STARTdone := FALSE; gCycle := 1; gTargetAngle := gDefaultTarget (только авто-режим)
2STOPdone := TRUE; autoBend := FALSE
3ZEROОбнулить энкодер, done := FALSE, gCycle := 1
4AUTO_BENDautoBend := TRUE (авто-режим, NOT done)
6SON_ONsonZ := TRUE; sonC := TRUE; done := FALSE; autoBend := FALSE; gCycle := 1 → DO7/DO8 = ON. Блокируется для оси с активным алармом (almZ/almC)

Важно: команды SON/E-STOP передаются как level-команды (без короткого импульса), чтобы ПЛК не пропускал их при джиттере опроса Modbus.

5. Аварийные сигналы серво (ALM) — D17/D18

5.1. Цепочка аварийного сигнала

MR-JE-200A ALM (CN1 pin 37, бело-голубой провод, b-contact NC)
  → ПЛК Owen DI: D17 (gDI_17 AT %IX1.1.6) = Серво Z ALM
                    D18 (gDI_18 AT %IX1.1.7) = Серво C ALM
  → Modbus регистр 7 (gAlarmFlags): bit0 = almZ, bit1 = almC
  → Teensy 4.1 (читает рег. 7)
  → USB-Serial JSON: {"alm":{"z":0,"c":0}}
  → EXE TubeBender (отображение + защита)

5.2. Поведение b-contact (выход ALM серводрайвера MR-JE-200A)

По умолчанию выход ALM серводрайвера MR-JE-200A (CN1 pin 37) настроен как b-contact (нормально замкнутый, NC):

Состояние сервоТранзистор выхода ALMУровень на DI ПЛКИнтерпретация
Норма (нет аварии)Включён (ON)TRUE (лог. 1)Нормальная работа
Авария (AL.xx)Выключен (OFF)FALSE (лог. 0)Авария серво!

5.3. Логика защиты в ПЛК

ПЛК детектирует аварию инверсией входа (b-contact):

almZ := NOT gDI_17;   (* DI=FALSE → almZ=TRUE → авария серво Z *)
almC := NOT gDI_18;   (* DI=FALSE → almC=TRUE → авария серво C *)

При обнаружении аварии ПЛК выполняет:

  • Снятие SON (Servo ON) для аварийной оси: sonZ := FALSE / sonC := FALSE
  • Остановка автоматического цикла: autoBend := FALSE
  • Установка флага завершения: done := TRUE
  • CMD 6 (SON ON) блокируется для оси с активным алармом

5.4. Обработка аларма в Teensy и EXE

Teensy читает регистр 7 (gAlarmFlags) из Modbus и добавляет в JSON-статус поле "alm":{"z":0,"c":0} (0 = норма, 1 = авария). При обнаружении аларма Teensy останавливает оси и снимает SON.

EXE TubeBender парсит alm.z и alm.c из JSON и отображает «АЛАРМ серво Z/C! (D17/D18)» красным текстом в окнах «Подключение» и «Станок». Если аларм возникает во время выполнения программы, конечный автомат (MachPhase) переходит в фазу Error.
